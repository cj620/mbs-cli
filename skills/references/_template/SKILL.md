# <domain> — <中文模块名>

通过 `mbs <domain>` 命令查询<业务说明，例如：订单、采购单、财务报表>数据。

## 适用场景

> 一句话说明：什么角色、什么场景使用此模块。
> 例如："采购人员通过此模块查询采购单状态、供应商信息及采购报表。"

## 不适用场景

> 说明哪些意图不属于本模块，帮助 Agent 负向过滤，避免误路由。
> 例如："销售订单查询请用 orders 模块；财务结算请用 finance 模块。"

## 意图匹配

| 用户说 | 应调用 |
|--------|--------|
| "..." | `<domain> <action>`（需 xxx 参数） |
| "..." | `<domain> <action>`（需 xxx + yyy） |

## 命令一览

| 命令 | 说明 | 必填参数 |
|------|------|---------|
| `mbs <domain> <action>` | ... | `--param` |

## 上下文传递（如有层级依赖）

| 命令 | 提取字段 | 传入下一步 |
|------|---------|-----------|
| `<domain> <action>` | `data[].id` → someId | `--someId` |

## 典型场景

**场景描述：**
```bash
mbs <domain> <action> --param value
mbs <domain> <action2> --id <id from above>
```

## 注意事项

- 公司 ID：`1`=胤元，`33`=启元（如适用）
- 多个 ID 统一用逗号分隔，例如 `--ids A001,A002`
- 退出码说明：见 [../global.md](../global.md#退出码)
